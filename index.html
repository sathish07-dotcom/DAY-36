Local Storage & Global Storage (Session Storage)

In JavaScript, the concepts of **local storage** and **global storage** usually refer to different storage mechanisms that allow you to store data on the user's browser, with different scopes, lifetimes, and use cases.

Local Storage

Local Storage is part of the **Web Storage API** and allows web applications to store data in the browser. The data stored in Local Storage persists even after the browser is closed, and remains accessible for as long as the user does not explicitly delete it. It is **persistent**, **client-side**, and **synchronous**.

Key characteristics:

  • **Persistent storage**: Data does not expire and is available across sessions.
  • **Storage limit**: Typically around 5MB per origin (depends on the browser).
  • **Scope**: Data is accessible only within the same origin (same domain, protocol, and port).
  • **Synchronous API**: Access to local storage is synchronous (blocking).

    • // Set an item in Local Storage
    localStorage.setItem('username', 'JohnDoe');
    // Get an item from Local Storage
    let username = localStorage.getItem('username');
    console.log(username); // "JohnDoe"
    // Remove an item from Local Storage
    localStorage.removeItem('username');
    // Clear all items from Local Storage
    localStorage.clear();

  • **When to use Local Storage**: Local Storage is ideal for storing small amounts of data that you want to persist across sessions (like user preferences, theme settings, etc.).

    • Global Storage (Session Storage)

    Global Storage is a term that might sometimes be used to refer to **Session Storage**, though the terms can vary depending on context. **Session Storage** is also part of the **Web Storage API**, and it allows you to store data that is only available for the duration of the page session (i.e., as long as the browser tab is open). Once the tab is closed, the session data is deleted.

    Key characteristics:

  • **Temporary storage**: Data is lost once the tab or browser window is closed.
  • **Storage limit**: Typically around 5MB per session (depends on the browser).
  • **Scope**: Data is only accessible within the same window/tab, and cannot be shared between tabs or windows.
  • **Synchronous API**: Like Local Storage, access to Session Storage is synchronous.

    • // Set an item in Session Storage
    sessionStorage.setItem('sessionUser', 'JaneDoe');
    // Get an item from Session Storage
    let sessionUser = sessionStorage.getItem('sessionUser');
    console.log(sessionUser); // "JaneDoe"
    // Remove an item from Session Storage
    sessionStorage.removeItem('sessionUser');
    // Clear all items from Session Storage
    sessionStorage.clear();

  • **When to use Session Storage**: Session Storage is perfect for temporary data that only needs to exist while the user is on a specific page (e.g., form data during the user's session).

    • Key Differences between Local Storage and Session Storage

    Feature Local Storage ssion Storage
    **Persistence** Data persists across sessions (until manually deleted) Data is cleared once the browser tab is closed
    **Storage limit** Usually around 5MB per origin Usually around 5MB per session
    **Scope** Accessible from any window or tab with the same origin Accessible only within the same window or tab
    **Use case** For storing data that should persist across sessions (e.g., user preferences) For storing temporary data during a single session (e.g., form data)
    **Expiration** Does not expire, stays until deleted Expired once the tab is closed

    **Local Storage**:

    // Storing a user's settings in localStorage
    localStorage.setItem('theme', 'dark');

    // Storing temporary form input in sessionStorage
    sessionStorage.setItem('formName', 'John');
    sessionStorage.setItem('formEmail', 'john.doe@example.com');
